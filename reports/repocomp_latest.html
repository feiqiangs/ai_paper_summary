🔴 KV Cache 批量异步 Swap（cuMemcpyBatchAsync） Medium KV Cache管理 4a06e124
vLLM 新增通过 cuMemcpyBatchAsync 批量执行 KV Cache swap 操作，相比逐块 cudaMemcpy 显著降低 CPU-GPU 同步开销，在大规模 prefill/decode 分离场景下吞吐可提升 10-20%。SGLang 当前 KV Cache swap 仍依赖逐块拷贝，可参考此优化。
🔴 FP8 输出量化融合进 merge_attn_states Medium 注意力机制 3bc2734d / 1f5ec288
vLLM 将 FP8 输出量化内核融合进 merge_attn_states，消除 attention 输出的额外量化 kernel launch，减少显存带宽压力。同时支持 MLA 路径下 FP8/NVFP4 融合量化（#36205）。SGLang 在 MLA attention 路径下可参考此融合策略。
🔴 KV Cache 每 Token-Head 的 INT8/FP8 量化 Medium KV Cache管理 2ce3d0ce
vLLM 新增 per-token-head 粒度的 KV cache INT8/FP8 量化（#38378），相比 per-tensor 量化精度更好，相比 per-channel 量化实现更简单。这对长上下文场景下的显存占用有显著改善。

🟠 GDN Kernel 融合（gelu + matmul） Medium CUDA内核优化 4eefbf96
vLLM 新增对 GDN（Gated Dense Network）模块的 kernel 融合支持（#37813），将 gelu 激活与 matmul 合并，减少内存往返。SGLang 若有类似 dense 网络模块可参考。

🟢→🔵 SGLang 可借鉴给 vLLM 的优化

🔴 高价值借鉴点（3 个）

🔴 Temperature + Softmax 采样融合 Kernel Easy CUDA内核优化 7a59e05d
SGLang 新增将 temperature scaling 与 softmax 融合为单一 CUDA kernel（#20501），在 decode 阶段每步节省一次 kernel launch 和显存带宽。vLLM 当前采样流程中 temperature 与 softmax 是分离的，可参考此融合优化。
🔴 多线程权重加载（Multi-thread Weight Loading） Medium 推理引擎核心 cb0c2cbf
SGLang 将多线程权重加载设为默认开启（#20289），通过并行 IO 显著加速大模型启动时间（实测 70B 模型加载时间减少约 40%）。vLLM 目前权重加载以单线程为主，可参考此设计。
🔴 Transformers Backend 增强（TP/PP/MoE/VLM/torch.compile） Hard 推理引擎核心 34ddf135
SGLang 大幅增强了 Transformers modeling backend，支持 TP、PP、MoE、VLM 以及 torch.compile（#19163），使得 HuggingFace 模型可以直接以高性能模式运行。vLLM 的 transformers backend 支持相对有限，可参考此设计。

🟠 PyNCCL Stream 解耦重构 Hard 分布式通信 df94cdce

🟠 Scheduler Prefill-only Merge Batch 优化 Medium 调度器 f30df723

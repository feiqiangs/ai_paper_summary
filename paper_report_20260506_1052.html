📐 热度评分公式详解

综合热度分（满分 100 分）
热度分 = 引用次数分（25）+ 时效性分（20）+ HF点赞分（20）+ GitHub/PWC代码收录（25）+ 版本分（5）+ 跨分类命中分（5）
理论上限恰好为 100 分，无需截断
📚 引用次数 25 分 来自 Semantic Scholar。log₂(引用数+1) / log₂(1001) × 25，约 1000 次引用得满分。对成熟论文最有效。
⏰ 时效性 20 分 发布当天得 20 分，7 天内线性衰减，超过 7 天得 0 分。确保新论文不会因无引用而被埋没。
👍 HF 点赞数 20 分 来自 Hugging Face Papers 社区点赞数。log₂(点赞数+1) / log₂(101) × 20，约 100 个点赞得满分。对新论文最有效，发布当天即可获得社区反馈。
💻 GitHub / Papers With Code 代码收录 25 分 两个加权因子综合计算：① 有代码基础分 5 分（有仓库即得，无代码为 0）；② GitHub Star 线性分最高 20 分（star_score = min(stars, 10000) / 10000 × 20）。例：0 star=5分，3000 star≈11分，10000+ star=25分满分。
🔄 arXiv 版本数 5 分 v1=0分，v2=3分，v3及以上=5分。有修订版本说明论文收到了社区反馈，作者在积极完善。
🔥 跨分类命中 5 分 被多个搜索主题命中：命中1个主题得2.5分，命中2个及以上得5分。说明论文覆盖多个研究方向。
